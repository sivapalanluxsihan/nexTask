# nexTask — API Documentation

This document provides a summary of all REST API endpoints available in the nexTask backend. For interactive testing and full request/response schemas, use the **Swagger UI** at:

### 👉 [http://localhost:3000/api-docs](http://localhost:3000/api-docs)

The Swagger documentation is auto-generated by **TSOA** from TypeScript controller decorators and shared type interfaces.

---

## Standardized Response Format

All endpoints return a consistent JSON response structure:

### Success Response
```json
{
  "success": true,
  "message": "Descriptive success message",
  "data": { ... },
  "errors": null
}
```

### Error Response
```json
{
  "success": false,
  "message": "Descriptive error message",
  "data": null,
  "errors": {
    "field.name": "Specific field validation error"
  }
}
```

### HTTP Status Codes

| Code | Meaning | When Used |
|------|---------|-----------|
| `200` | OK | Successful retrieval or update |
| `201` | Created | Resource successfully created |
| `400` | Bad Request | Business rule violation, invalid token |
| `401` | Unauthorized | Missing/invalid/expired JWT token |
| `403` | Forbidden | Insufficient permissions, deactivated account |
| `404` | Not Found | Resource does not exist |
| `422` | Unprocessable Entity | Zod/TSOA validation failure |
| `500` | Internal Server Error | Unexpected server error |

## Authentication

All protected endpoints require a JWT Bearer token in the `Authorization` header:

```
Authorization: Bearer <jwt-token>
```

Obtain a token via `POST /auth/login`.

## WebSocket Events (Socket.IO)

### Connection

```javascript
const socket = io('http://localhost:3000', {
  auth: { token: 'your-jwt-token' }
});
```

### Client → Server Events

| Event | Payload | Description |
|-------|---------|-------------|
| `join-project` | `{ projectId }` | Join a project room |
| `leave-project` | `{ projectId }` | Leave a project room |
| `message:send` | `{ projectId, content, attachments? }` | Send a chat message |

### Server → Client Events

| Event | Payload | Description |
|-------|---------|-------------|
| `task:created` | `Task` | A task was created in the project |
| `task:updated` | `Task` | A task was updated |
| `task:deleted` | `{ taskId }` | A task was deleted |
| `comment:created` | `Comment` | A comment was posted |
| `comment:deleted` | `{ commentId, taskId }` | A comment was deleted |
| `assignment:updated` | `{ taskId, assignees }` | Task assignees changed |
| `message:new` | `Message` | New chat message received |

## RBAC Scope Reference

| Scope | Required Role | Description |
|-------|--------------|-------------|
| `global:admin` | `ADMIN` | System-wide administrative access |
| `global:pm` | `ADMIN` or `PROJECT_MANAGER` | Can create/manage projects |
| `project:member` | Any member of the project | Can view project resources |
| `project:manager` | Project owner or PM member | Can modify project resources |
